1. Vision du Projet
Cette application Laravel combine trois fonctionnalités majeures dans une interface élégante glassmorphism : la gestion de recettes avec calcul nutritionnel automatique, la recherche intelligente par ingrédients disponibles (contenu du frigo), et l'analyse d'images IA pour identifier les aliments et estimer leurs calories.

Recherche par ingrédients
Entrer les ingrédients du frigo → recettes faisables triées par correspondance

Service Edamam Food Database API
Créer NutritionService.php. Endpoint: https://api.edamam.com/api/food-database/v2/parser?ingr={ingredient}&app_id={ID}&app_key={KEY}. Méthode getCalories(string $ingredient): array retourne calories, protein, fat, carbs, fiber. Cache Laravel 30 min (Redis ou file) pour limiter les requêtes API. Fallback automatique sur OpenFoodFacts API (gratuite, sans clé).
public function getCalories(string $ingredient): array {
  return Cache::remember('nutrition_'.$ingredient, 1800, function() use ($ingredient) {
    $response = Http::get('https://api.edamam.com/api/food-database/v2/parser', [
      'ingr' => $ingredient, 'app_id' => env('EDAMAM_APP_ID'), 'app_key' => env('EDAMAM_APP_KEY')
    ]);
    return $response->json()['hints'][0]['food']['nutrients'] ?? $this->fallback($ingredient);
  });
}

// Scoring SQL
$recipes = Recipe::select('recipes.*',
  DB::raw('COUNT(DISTINCT i.id) / r_total.total * 100 AS match_percent'))
  ->join('ingredients as i', 'i.recipe_id', 'recipes.id')

4. Extraits de Code Essentiels
4.1 ClaudeVisionService.php — Appel API Vision
<?php namespace App\Services;
use Illuminate\Support\Facades\Http;
use Illuminate\Support\Facades\Log;
class ClaudeVisionService {
  private string $apiKey;
  private string $model = 'claude-opus-4-6';
  public function __construct() {
    $this->apiKey = env('ANTHROPIC_API_KEY');
  }
  public function analyzeFood(string $imagePath): array {
    $imageData = base64_encode(file_get_contents(storage_path('app/public/'.$imagePath)));
    $mimeType  = mime_content_type(storage_path('app/public/'.$imagePath));
    $response = Http::timeout(30)->withHeaders([
      'x-api-key'        => $this->apiKey,
      'anthropic-version'=> '2023-06-01',
      'content-type'     => 'application/json',
    ])->post('https://api.anthropic.com/v1/messages', [
      'model'      => $this->model,
      'max_tokens' => 1000,
      'messages'   => [[
        'role'    => 'user',
        'content' => [
          ['type' => 'image', 'source' => ['type' => 'base64',
           'media_type' => $mimeType, 'data' => $imageData]],
          ['type' => 'text', 'text' => $this->getFoodPrompt()],
        ]
      ]]
    ]);
    $content = $response->json()['content'][0]['text'] ?? '{}';
    return json_decode($content, true) ?? [];
  }
}

4.3 AlpineJS — Recherche Multi-Tags (Frigo)
<div x-data="{ tags: [], input: '' }">
  <div class='tag-container'>
    <template x-for='(tag, i) in tags' :key='i'>
      <span class='tag'>
        <span x-text='tag'></span>
        <button @click='tags.splice(i,1)'>x</button>
      </span>
    </template>
    <input x-model='input' @keydown.enter.prevent="if(input.trim()) { tags.push(input.trim()); input=''; }"
           @keydown.backspace="if(!input && tags.length) tags.pop()"
           placeholder='Ajouter un ingrédient...' />
  </div>
  <button @click="$fetch('/search', {method:'POST', body: JSON.stringify({ingredients: tags})})">
    Trouver des recettes
  </button>
</div>

Inscription gratuite sur developer.edamam.com. Le plan gratuit offre 1 000 requêtes par mois, largement suffisant pour le développement. Créer une application de type 'Food Database API' pour obtenir les identifiants.
